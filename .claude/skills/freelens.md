# Freelens Project Skill

You are working on **Freelens** — a free, open-source Electron-based Kubernetes UI.
Repo: `~/dev/oss/freelens` (fork of OpenLens, upstream: `freelensapp/freelens`)

## Dev Workflow

```bash
# Switch to required Node version first
node-22               # homebrew shell function, switches to node@22

# Install & build
pnpm i
pnpm build:di         # REQUIRED: generate DI registration files
pnpm build            # build all packages via Turbo

# Run app
pnpm start

# After any .injectable.ts change:
pnpm build:di && pnpm build && pnpm start

# Tests & lint
pnpm test:unit:core   # core package unit tests (fastest)
pnpm test:unit        # all unit tests
pnpm lint             # Biome + Trunk
pnpm lint:fix         # auto-fix

# Clean rebuild (when Turbo cache causes stale state)
rm -rf .turbo packages/core/dist freelens/dist
pnpm build:di && pnpm build
```

## DI System (Critical)

Uses `@ogre-tools/injectable`. **Never manually edit `register-injectables.ts` files** — they are auto-generated.

**Rules:**
- Injectable files must have `.injectable.ts` suffix
- Run `pnpm build:di` after any add/move/rename of `.injectable.ts` files
- Registration hierarchy: leaf → aggregator → root (`register-injectables-main.ts` / `register-injectables-renderer.ts`)
- Shared aggregators only aggregate `common/` subdirectories — NOT `main/` or `renderer/`

**Common DI errors:**
- `Tried to register same injectable multiple times` → re-run `pnpm build:di`
- `Tried to inject non-registered injectable` → DI files not regenerated

## Adding a New User Preference Setting (step-by-step)

1. **Descriptor** — add to `packages/core/src/features/user-preferences/common/preference-descriptors.injectable.ts`:
   ```typescript
   myNewPref: getPreferenceDescriptor<string | undefined>({
     fromStore: (val) => val,
     toStore: (val) => val || undefined,
   }),
   ```
   `UserPreferencesModel` and `UserPreferencesState` are auto-typed from this — no separate changes needed.

2. **Storage wiring** — add to `packages/core/src/features/user-preferences/common/storage.injectable.ts`:
   - In `fromStore` action: `state.myNewPref = descriptors.myNewPref.fromStore(preferences.myNewPref);`
   - In `toJSON` return object: `myNewPref: descriptors.myNewPref.toStore(state.myNewPref),`

3. **UI component** — create in `packages/core/src/features/preferences/renderer/preference-items/kubernetes/kubectl/<feature-name>/`:
   - `<feature-name>.tsx` — React component using `withInjectables`, `observer`, `SubTitle`, and an input/select
   - `<feature-name>-preference-block.injectable.ts` — registers with `preferenceItemInjectionToken`, sets `parentId: "kubectl"`, `orderNumber: <N>`
   - `register-injectables.ts` — AUTO-GENERATED by `pnpm build:di` (do not create manually)

4. **Run `pnpm build:di`** — auto-creates `register-injectables.ts` and updates parent aggregator.

## PR Workflow

Before creating an upstream PR, always run:
```bash
.claude/scripts/prep-pr.sh
```
This verifies no `.claude/` files are staged, runs lint, and runs core unit tests.

The `.claude/` directory is tracked locally as experiment documentation but must **never** appear in upstream PRs.
